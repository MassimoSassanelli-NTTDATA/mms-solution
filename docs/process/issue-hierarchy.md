# Issue Hierarchy

Epic -> Story -> Sub-Issue / Implementation Task -> Pull Request

- Epics are created in the platform repository (`type:epic`).
- Stories are created in the platform repository (`type:story`).
- Implementation tasks are created in the affected code repositories (`type:task`, `repo:<name>`).

Each issue carries exactly one `status:*` label that reflects its position in the
process. See [label-model.md](label-model.md) for the status flow and automation.
